# Система бонусов (статы) — гайд для геймдизайнера

Как настраивать прибавки к параметрам игры через **Modifiers** (список `StatModifier`). Настраивается в инспекторе Unity, кода не требует.

Сейчас в системе три бонуса: **ProductionGlobal**, **ResourceYield**, **UnitSpeed**. Ниже — как каждый работает, с какими параметрами, и где грабли.

---

## Коротко

- **База** параметров лежит в конфигах (скорость юнита, сколько воркер добывает и т.д.) и не меняется.
- **Бонусы** — отдельный слой прибавок. Ты добавляешь «бумажки» (`StatModifier`), они складываются по формуле и меняют итог.
- Один `StatModifier` = один бонус к одному параметру.

Бонусы задаются списком в:
- **`AgeDef` → Modifiers** — бонусы эпохи (основное);
- **`RunManager` → Debug Start Modifiers** — для теста в редакторе;
- (позже — перки, мета-апгрейды: тот же `StatModifier`).

---

## Поле StatModifier: 5 параметров

| Поле | Что это |
|---|---|
| **Id** | какой бонус (ProductionGlobal / ResourceYield / UnitSpeed) |
| **Unit Scope** | к какому типу юнита применять (Farmer, Lumberjack, Miner…) |
| **Resource Scope** | к какому ресурсу применять (Food, Wood, Stone, Metal, Coins) |
| **Flat** | плоская прибавка к базе |
| **Percent** | процентная прибавка (`0.5` = +50%) |

⚠️ **Не для каждого Id важны оба scope-поля.** Лишние поля просто игнорируются (см. таблицу ниже). Это главный источник путаницы.

---

## Как считается итог

Одна формула на всё:

```
итог = (база + сумма Flat) × (1 + сумма Percent)
```

- **Flat** прибавляется к базе (в единицах параметра).
- **Percent** — доля: `0.5` = +50%, `1` = +100% (×2), `-0.5` = −50%.
- Несколько бонусов одного вида **складываются**: два `+50%` дают `+100%` (×2), **не** ×2.25.

---

## Таблица: какие поля работают у какого бонуса

| Id | Unit Scope | Resource Scope | Flat | Percent | На что влияет |
|---|---|---|---|---|---|
| **ProductionGlobal** | ❌ игнор | ❌ игнор | ✅ | ✅ | вся добыча ресурсов (глобально) |
| **ResourceYield** | ✅ **нужен** | ✅ **нужен** | ✅ | ✅ | сколько воркер добывает конкретного ресурса |
| **UnitSpeed** | ✅ **нужен** | ❌ игнор | ✅ | ✅ | скорость передвижения юнита |

- ❌ игнор — можешь оставить любым, роли не играет.
- ✅ нужен — **обязан совпасть** с реальностью, иначе бонус молча не применится.

---

## 1. ProductionGlobal — глобальный множитель добычи

Умножает **всю** добычу ресурсов (любой воркер, любой ресурс).

- **Unit Scope / Resource Scope:** игнорируются.
- **Percent:** основной инструмент (`0.2` = +20% ко всей добыче).
- **Flat:** прибавит фикс. число к **каждому** сбору (обычно не нужно, ставь 0).

**Пример** — «+20% ко всей добыче»:
```
Id = ProductionGlobal
Percent = 0.2
(scope-поля не важны)
```

---

## 2. ResourceYield — добыча конкретного ресурса конкретным воркером

Меняет, сколько воркер добывает за один удар. База берётся из источника (`ResourceSourceDef → Worker Yields`).

- **Unit Scope:** тип воркера, который добывает (обязателен).
- **Resource Scope:** **ПРОИЗВОДИМЫЙ ресурс**, не название источника (обязателен).
- **Flat:** +N ресурса за удар. **Percent:** % к добыче.

### ⚠️ Главная грабля: Resource Scope = что ПРОИЗВОДИТСЯ, а не имя источника

«Пшеница» (Wheat) даёт ресурс **Food**, а не «Wheat». Если поставить Resource Scope во что-то, чего воркер реально не добывает, — бонус не сработает.

### Какие пары (воркер → ресурс) сейчас существуют

| Источник | Ресурс (Resource Scope) | Воркеры (Unit Scope) → база за удар |
|---|---|---|
| Wheat | **Food** | Farmer → 1 |
| Tree | **Wood** | Farmer → 1, Lumberjack → 2 |
| Smithy | **Metal** | Farmer → 1, Miner → 2 |
| Market | **Coins** | Farmer → 1, Miner → 2 |

> Бонус на пару, которой нет в таблице (например Miner + Food), ничего не даст — такой добычи не существует. Уточняй актуальные пары в ассетах `ResourceSourceDef`.

**Пример** — «Фермеры добывают на 50% больше еды»:
```
Id = ResourceYield
Unit Scope = Farmer
Resource Scope = Food
Percent = 0.5
```

**Пример** — «Лесорубы: +1 дерева за удар»:
```
Id = ResourceYield
Unit Scope = Lumberjack
Resource Scope = Wood
Flat = 1
```

### Нет «всех ресурсов» одной бумажкой

Resource Scope — всегда конкретный ресурс. Чтобы усилить воркера по нескольким ресурсам, добавь **по одному `StatModifier` на каждый ресурс**. Нужен глобальный буст всей добычи — это `ProductionGlobal`.

---

## 3. UnitSpeed — скорость юнита

Меняет скорость передвижения. База — `UnitDef → Speed`.

- **Unit Scope:** тип юнита (обязателен).
- **Resource Scope:** игнорируется.
- **Flat:** +N к скорости. **Percent:** % к скорости.

**Пример** — «Фермеры на 30% быстрее»:
```
Id = UnitSpeed
Unit Scope = Farmer
Percent = 0.3
```

> Как и с ресурсами: «все юниты» одной бумажкой нельзя — по одному модификатору на тип юнита.
> Скорость пересчитывается при следующем вылете юнита из здания (не мгновенно на уже летящем).

---

## На что бонусы влияют и НЕ влияют (границы системы сейчас)

**Влияют:**
- добыча ресурсов (ProductionGlobal, ResourceYield);
- скорость юнитов (UnitSpeed).

**Пока НЕ настраиваются бонусами** (нет соответствующего Id — добавляет программист):
- вместимость домов (сколько воркеров в здании);
- скорость перезарядки зданий;
- ускорение при вылете / буст от тапа;
- стоимость построек, возврат при продаже.

**Бонусы на добычу применяются ТОЛЬКО к добыче.** Трата ресурсов, возврат при продаже и прочие «сырые» начисления не бустятся (ProductionGlobal их не касается).

## Рецепты (частые бонусы → точные значения)

| Хочу | Id | Unit | Resource | Flat | Percent |
|---|---|---|---|---|---|
| +20% ко всей добыче | ProductionGlobal | — | — | 0 | 0.2 |
| Фермеры +50% еды | ResourceYield | Farmer | Food | 0 | 0.5 |
| Лесорубы +1 дерева/удар | ResourceYield | Lumberjack | Wood | 1 | 0 |
| Шахтёры +25% металла | ResourceYield | Miner | Metal | 0 | 0.25 |
| Шахтёры +25% и металла, и монет | ResourceYield ×2 | Miner | Metal / Coins | 0 | 0.25 |
| Фермеры +30% скорости | UnitSpeed | Farmer | — | 0 | 0.3 |

---

## Как складываются бонусы (стэкинг)

- **Одинаковые** (один Id + одинаковые scope) — Flat суммируются, Percent суммируются.
  Две эпохи по `+50%` ProductionGlobal → итог `×2.0`.
- **Разные виды** на одну добычу — перемножаются.
  Farmer, Food, база 1, при `ResourceYield +50%` и `ProductionGlobal +20%`:
  `(1 × 1.5) × 1.2 = 1.8` за удар.

---

## Если бонус «не работает» — чек-лист

1. **ResourceYield:** Resource Scope = **производимый** ресурс (Wheat = Food!), Unit Scope = реально добывающий воркер. Сверься с таблицей пар.
2. Бонус на добычу проверяется **на добыче** — трата/продажа не бустятся.
3. Модификатор должен быть в **применённом** списке: куплена эпоха с ним, либо он в `RunManager → Debug Start Modifiers`.
4. Смотри точное значение в **Console**: строка `[Resources] +1.5 Food …`. Полоска сверху округляет до целого — по ней бонус на мелких числах не виден.
5. Для UnitSpeed/ResourceYield лишние scope-поля не мешают, но **нужные** обязаны совпадать.

---

## Где заполнять в Unity

- **Эпоха:** ассет `AgeDef` → список **Modifiers** → `+` → задай Id / Unit Scope / Resource Scope / Flat / Percent.
- **Тест:** компонент `RunManager` → **Debug Start Modifiers** (применяется на старте рана; после теста очисти).

---

## Быстрая шпаргалка

- `итог = (база + ΣFlat) × (1 + ΣPercent)`; `Percent 0.5` = +50%.
- **ProductionGlobal** — scope не нужен, бустит всю добычу.
- **ResourceYield** — нужны ОБА scope; Resource = **производимый** ресурс (Wheat→Food); нет «всех ресурсов» одной бумажкой.
- **UnitSpeed** — нужен только Unit Scope.
- Одинаковые бонусы складываются, разные виды перемножаются.
- Бустится только добыча; траты/возвраты — нет.
- Проверяй в Console (точное дробное), не по округлённой полоске.
